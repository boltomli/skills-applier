## Docstring Structure

Generate a docstring with the following sections:

1. **Summary**: One-line description of what the function does

2. **Extended Description**: Detailed explanation (1-3 paragraphs) of:
   - What the function does
   - How it works
   - When to use it

3. **Parameters Section** (if applicable):
   ```
   Parameters
   ----------
   param1 : type
       Description of param1
   param2 : type
       Description of param2
   ```

4. **Returns Section** (if applicable):
   ```
   Returns
   -------
   return_type
       Description of return value
   ```

5. **Raises Section** (if applicable):
   ```
   Raises
   ------
   ValueError
       Description of when ValueError is raised
   ```

6. **Examples Section** (required):
   ```
   Examples
   --------
   >>> import numpy as np
   >>> data = np.array([1, 2, 3, 4, 5])
   >>> result = function_name(data)
   >>> print(result)
   ```

7. **Notes Section** (if applicable):
   ```
   Notes
   -----
   Important notes about the function
   ```

8. **References Section** (if applicable):
   ```
   References
   ----------
   .. [1] Author, "Title", Journal, Year.
   ```

## Guidelines

- Use NumPy docstring style
- Be concise but comprehensive
- Include type hints in parameter descriptions
- Provide realistic examples
- Document any assumptions or requirements
- Include warnings about common pitfalls

Return the docstring as plain text.
